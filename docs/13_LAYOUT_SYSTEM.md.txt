# 13 – Layout System

## Purpose

This document defines the spatial system used throughout the portfolio.

Spacing should create rhythm, hierarchy and breathing room.

Never reduce spacing simply to fit more content.

Whitespace is a design element.

---

# Layout Philosophy

The portfolio should feel open.

Large imagery deserves space.

Typography should never feel crowded.

Every section should have a clear beginning and end.

Negative space creates emphasis.

---

# Content Width

The portfolio should use a centered content container.

Maximum content width:

1600px

Very large imagery may intentionally extend beyond this limit when it strengthens the composition.

---

# Horizontal Padding

Desktop

64px

Tablet

40px

Mobile

24px

Maintain consistent page padding throughout the website.

---

# Vertical Rhythm

Major sections

160px top and bottom spacing.

Minor groups

64px spacing.

Related elements

24–32px spacing.

Small UI elements

8–16px spacing.

Always prefer generous spacing over compressed layouts.

---

# Grid

Desktop

12-column layout.

Tablet

8-column layout.

Mobile

Single-column layout.

The grid exists to create alignment, not limitation.

Breaking the grid is allowed when it improves the composition.

---

# Alignment

Default alignment is left.

Avoid center-aligning large blocks of text.

Titles should visually align with imagery.

Maintain strong vertical rhythm.

---

# Selected Works

Projects should feel like gallery pieces.

Large imagery.

Minimal supporting text.

Generous spacing between projects.

Never allow projects to feel crowded.

---

# Category Grid

Desktop

2 columns

Tablet

2 columns

Mobile

1 column

Cards should maintain equal proportions.

Spacing between cards should allow the textured background to remain visible.

---

# About Section

Content should breathe.

Allow generous spacing between images and text.

Avoid long uninterrupted paragraphs.

---

# Contact Section

Minimal.

Simple.

Clear.

No unnecessary decoration.

---

# Responsive Design

The experience should remain consistent across all screen sizes.

Hierarchy should never change.

Only the layout should adapt.

---

# Rule

When uncertain:

Choose more whitespace.

Never reduce spacing simply to fit additional content.
